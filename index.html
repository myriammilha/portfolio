Jeu Bataille Navale (Equipe Front-End)

Langages et Frameworks :

  • Front-end :
    • 🔹JavaScript (avec React.js) : pour créer l’interface utilisateur interactive.
    • 🔹CSS3 : pour le style et les animations visuelles.
  • Back-end :
    • 🔹Kotlin : pour coder la logique du jeu côté serveur.
    • 🔹SpringBoot : framework utilisé avec Kotlin pour créer l’API REST.

  • Développement d’une interface front-end interactive et immersive pour un jeu de bataille navale, jouable en solo ou en 1v1 en ligne.
    Le front-end, réalisé en React.js, intègre des animations, un brouillard de guerre, un système de score en temps réel, ainsi que des interactions avec une API back-end (en Kotlin/SpringBoot).
  • Conception de l’interface sur Canva, intégration avec React et animations dynamiques.
  • Drag-and-drop avec React-DnD, effets sonores interactifs via useSound.
  • Communication avec le backend en Spring Boot via API asynchrones (ES6).
  • Gestion collaborative du code avec Git.
  • - Le front-end (interface utilisateur) envoie la demande au back-end (logique métier) via une requête HTTP.
  • – Le back-end reçoit la demande et décide :
    1. Est-ce que l'action est possible ?
    2. Est-ce que l’attaque touche un bateau ?
    3. Est-ce que le joueur peut encore jouer ?
    4. Est-ce que la partie est finie ?
  • – Le back-end met à jour l’état du jeu.
  • – Le back-end il renvoie une réponse au front-end.
    1. Front-end → action utilisateur
    2. Back-end → décision, règles, mise à jour
    3. Front-end → affichage des résultats
